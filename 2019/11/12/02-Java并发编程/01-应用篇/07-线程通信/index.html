nofity和condition方式分别实现简易数据库连接池

数据库连接步骤:

  1. 加载驱动
  2. 获取连接
  3. 发送执行SQL
  4. 处理数据
  5. 关闭连接,释放资源

  普通客户端过来连接web程序,web程序对应连接数据库,用完之后销毁数据库连接.在做demo时看不出来性能,但是在高并发的情况下每次过来创建连接销毁连接,给数据库的压力非常大,性能损耗非常高
  所以在用户还没过来当容器开始启动的时候,建立与数据库的连接(比如100个—调优关键参数),当用户请求过来,就不再创建连接,这时就从池子中获取连接,操作数据库,请求结束之后把连接放回池子,原本操作一次数据库的时间=创建连接s1+操作数据库s2+释放连接s3;现在只剩s2,性能大大提高了
